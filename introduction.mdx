**Google Docs** is a documentation hosting platform built for teams who want their docs to live alongside their code. Connect your private git repository, push your content, and Google Docs takes care of the rest — building, hosting, and publishing your site automatically on every update. No servers to manage, no deployment pipelines to configure.

## What is Google Docs?

Google Docs is a documentation hosting platform that bridges the gap between your git repository and a polished, publicly shareable docs site. You write your documentation as MDX files directly in your repo. Every time you push a change, Google Docs detects it and automatically rebuilds and republishes your site — so your docs are always in sync with your code.

It's designed for engineering teams, developer-focused companies, and anyone who wants documentation that's version-controlled, easy to maintain, and effortless to deploy.

## How It Works

Getting your docs live with Google Docs is a four-step process:

1. **Write** your documentation as `.mdx` files inside your private git repository.
2. **Configure** a `docs.json` file at the root of your repo to define navigation, branding, and site settings.
3. **Push** your changes to your git remote as you normally would.
4. **Done** — Google Docs automatically detects the push, builds your site, and publishes the latest version.

There's no build command to run locally, no hosting account to set up, and no deployment workflow to maintain. Your repository is the single source of truth.

<Note>
  Your documentation is private by default — only people you share the link with can access it.
</Note>
