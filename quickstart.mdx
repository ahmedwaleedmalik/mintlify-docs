title: "Get Started with Google Docs: Publish Docs in Minutes"

---

This guide walks you through publishing your documentation site for the first time. By the end, you'll have a live docs site connected to your private git repository and updating automatically on every push.

<Steps>
  <Step title="Set up your repository">
    Your documentation lives as `.mdx` files inside a git repository. If you're starting from scratch, create a new repo and add the following files at the root level:

    ```text
    my-docs/
    ├── docs.json
    ├── index.mdx
    └── quickstart.mdx
    ```

    You can organize pages into subdirectories as your docs grow — but a flat structure like this is all you need to get started.
  </Step>
  <Step title="Create your docs.json">
    The `docs.json` file at the root of your repo controls your site's navigation, branding, and settings. Create one now with a minimal configuration:

    ```json docs.json
    {
      "$schema": "https://mintlify.com/docs.json",
      "name": "My Docs",
      "navigation": {
        "groups": [
          {
            "group": "Get Started",
            "pages": ["index", "quickstart"]
          }
        ]
      }
    }
    ```

    The `navigation` field defines the sidebar structure your readers will see. Each entry in `pages` maps to an `.mdx` file by its path relative to the repo root (without the file extension).
  </Step>
  <Step title="Write your first page">
    Every page starts with a frontmatter block that sets its title and description. Here's a minimal example for `index.mdx`:

    ```mdx index.mdx
    ---
    title: "Welcome"
    description: "Welcome to my documentation."
    ---

    Welcome! This is my documentation site.
    ```

    You can use any standard MDX content below the frontmatter — headings, code blocks, callouts, and all Mintlify components are supported out of the box.
  </Step>
  <Step title="Push and publish">
    Once your files are in place, commit and push your changes to your git remote:

    ```bash
    git add .
    git commit -m "Initial docs"
    git push
    ```

    Google Docs detects the push and automatically builds and deploys your site. Your documentation will be live at your Google Docs URL within moments — no further action required.
  </Step>
</Steps>

<Tip>
  Use the `mint dev` CLI command locally to preview your docs before pushing. Run it from the root of your repo and open `http://localhost:3000` to see a live preview that hot-reloads as you edit.
</Tip>
